Seperate an Integer into seperate Digits

var n =  123456789;
          var digits = (""+n).split("");
          console.log(digits); //[ '1', '2', '3', '4', '5', '6', '7', '8', '9' ]

Locate a Substring within a String

Using indexOf()

var stringOne = "Johnny Waldo Harrison Waldo";
          var wheresWaldo = stringOne.indexOf("Waldo"); // 7

Using lastIndexOf():

var stringOne = "Johnny Waldo Harrison Waldo";
          var wheresWaldo = stringOne.lastIndexOf("Waldo"); // 22

Replace a Substring

Replacing a single string with another string

var slugger = "Josh Hamilton";
          var betterSlugger = slugger.replace("h Hamilton", "e Bautista");
          console.log(betterSlugger); // "Jose Bautista"

Replacing all occurance of a string with another string.
- Using Regexp

var myString = "She sells automotive shells on the automotive shore";
          var newString = myString.replace(/automotive/g, "sea");
          console.log(newString); // "She sells sea shells on the sea shore"

Function Expressions

Function Expression is building functions within code execution rather than at program load time. The function keyword can be used to define a function inside an expression.
In the below code "function" is a function expression and it loads only when this particular line of code is reached inside your program. Since we are using an assignment statement to assign a function to a variable. We need a semicolon to complete the assignment statement in a file "};".
The function name can be omitted, in which case the function is Anonymous Function. 

var diff = function(a,b)
          {
          return a-b;
          };

Returning a function and immediately invoking it

function adventureSelector(userChoice) {
  if (userChoice == 1) {
    return function() {
      alert("You selected the Vines of Doom!");
    };
  } else if (userChoice == 2) {
    return function() {
      alert("Looks like you want the Lake of Despair!");
    };
  } else if (userChoice == 3) {
    return function() {
      alert("The Caves of Catastrophe!");
    };
  }
}

then immediately invoking this funtion

adventureSelector(3)();

Load Order in Javascript

work in progress...

any variable declarations get moved to the top of the load, and are initially undefined. These are followed by any function declarations in the order they are declared. Any code that is executed, like assignments, operations, or function calls, are always last and proceed from first appearance in the code body to the last. Also, when a function is loaded into memory after a function that has the same name, the latter function overwrites the original one.
